- **Rota `/:userId/reviews` vs `/:id`**: A ordem atual funciona porque Express casa rotas exatas primeiro. `GET /profile/mine` e `GET /admin/reports` são literais e casam antes de `/:userId/reviews`. - **`submitReport` no frontend**: A URL `/api/providers/${providerId}/report` casa com o backend `POST /:userId/report`. Mas o `providerId` passado pelo frontend é o `id` da URL (`/prestador/:id`) que é o `user_id` — o backend resolve corretamente via `WHERE user_id = ?`. Funciona, mas o nome `providerId` no frontend é enganoso (é um `userId`).
